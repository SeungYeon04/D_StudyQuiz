### 1. 소프트웨어 공학

#### 1. UML(Unified Modeling Language) diagram

통합 모델링 언어, 소프트웨어  
시스템을 시각적으로 표현.  
  
> [구조 다이어그램]
> 1. 클래스 다이어그램
> 정적 관계 설명 / 클래스, 속성, 연산 등 표시
> 2. 객체 다이어그램
> 특정 시점의 객체(인스턴스)와 그 관계
> 3. 컴포넌트 다이어그램
> 시스템을 구성하는 컴포넌트(모듈)의 구조
> 4. 배치 다이어그램
> 시스템의 물리적인 배치
> 5. 패키지 다이어그램
> 클래스, 컴포넌트(모듈)를 패키지 단위로 표현
> 
> [행위 다이어그램]
> 1. 유스 케이스 다이어그램
> 사용자와 시스템 기능 간의 상호작용
> 2. 시퀀스 다이어그램
> 객체 간 메시지 흐름과 시간 순서 표현
> 3. 활동 다이어그램
> 프로그램의 흐름 (조건문, 반복문 등)
> 4. 상태 다이어그램
> 객체의 상태 변화 표현
> 5. 커뮤니케이션 다이어그램
> 객체 간 메시지 중심
> 6. 타이밍 다이어그램
> 시간의 흐름에 따른 객체 상태 표현

#### 2. 응집도 / 결합도

[응집도 (Cohesion)]  
 하나의 모듈 내부 요소들이 얼마나  
밀접하게 연관되어 있는가 (높을수록 좋음)  
  
| 응집도 | 설명 | 
| :--- | :---: | 
| 우연적(Coincidental) | 관련 없음 | 
| 논리적(Logical) | 논리적으로 비슷한 작업(switch-case) | 
| 시간적(Temporal) | 시간적으로 동시에 실행 | 
| 절차적(Procedural) | 절차적으로 연관된 작업들(목적은 다름) | 
| 통신적(Communicational) | 동일한 데이터를 사용하는 작업 |
| 순차적(Sequential) | 한 작업의 결과를 다음 작업 입력 | 
| 기능적(Functional) | 하나의 기능 수행에 집중 |
  
[결합도 (Coupling)]  
모듈 간에 서로 얼마나 의존하는가 (낮을수록 좋음)  
  
| 결합도 | 설명 | 
| :--- | :---: | 
| 내용(Content) | 다른 모듈 내부를 직접 접근하거나 수정 |
| 공통(Common) | 전역 변수를 공유하여 사용 | 
| 외부(External) | 외부 시스템, 인터페이스 등에 의존 |
| 제어(Control) | 제어 정보를 전달 (플래그 등) |

스탬프(Stamp) 필요 이상의 데이터 전달 (a[1]만 필요한데 a 배열 전체 전달)
